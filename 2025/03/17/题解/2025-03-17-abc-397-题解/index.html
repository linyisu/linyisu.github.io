C - Variety Split Easy

题目大意

给定一个长度为 NNN 的整数序列：A=(A1,A2,…,AN)A = (A_1, A_2, \dots, A_N)A=(A1​,A2​,…,AN​)。

将 AAA 分成两个非空的连续子数组，求这两个子数组中不同整数的个数之和的最大值。

更正式地，给定一个整数 iii，其中 1≤i≤N−11 \leq i \leq N-11≤i≤N−1，要求求出如下两个值的和的最大值：

  1. 子数组 (A1,A2,…,Ai)(A_1, A_2, \dots, A_i)(A1​,A2​,…,Ai​) 中不同整数的个数。
  2. 子数组 (Ai+1,Ai+2,…,AN)(A_{i+1}, A_{i+2}, \dots, A_N)(Ai+1​,Ai+2​,…,AN​) 中不同整数的个数。

Constraints

  • 2≤N≤3×1052 \leq N \leq 3 \times 10^52≤N≤3×105
  • 1≤Ai≤N1 \leq A_i \leq N1≤Ai​≤N (1≤i≤N1 \leq i \leq N1≤i≤N)
  • All input values are integers.

思路

可以用两个数组 pre 和 bak s分别表示 从 1∼i1 \sim i1∼i 子段不同整数个数之和 和 从 i∼ni \sim ni∼n 子段不同整数个数之和。

通过上面的预处理, 我们便可以快速得知每个切割位置得到两个子段中不同整数个数和, 因此, 遍历每个可切割位点, 更新答案即可。

F - Variety Split Hard

题目大意

给定一个长度为 NNN 的整数序列：A=(A1,A2,…,AN)A = (A_1, A_2, \dots, A_N)A=(A1​,A2​,…,AN​)

将 AAA 分成三个非空的连续子数组，求这三个子数组中不同整数的个数之和的最大值。

更正式地，给定一对整数 (i,j)(i, j)(i,j)，其中 1≤i<j≤N−11 \leq i < j \leq N-11≤i<j≤N−1，要求求出如下三个值的和的最大值：

  1. 子数组 (A1,A2,…,Ai)(A_1, A_2, \dots, A_i)(A1​,A2​,…,Ai​) 中不同整数的个数。
  2. 子数组 (Ai+1,Ai+2,…,Aj)(A_{i+1}, A_{i+2}, \dots, A_j)(Ai+1​,Ai+2​,…,Aj​) 中不同整数的个数。
  3. 子数组 (Aj+1,Aj+2,…,AN)(A_{j+1}, A_{j+2}, \dots, A_N)(Aj+1​,Aj+2​,…,AN​) 中不同整数的个数。

Constraints

  • 3≤N≤3×1053 \leq N \leq 3 \times 10^53≤N≤3×105
  • 1≤Ai≤N1 \leq A_i \leq N1≤Ai​≤N (1≤i≤N1 \leq i \leq N1≤i≤N)
  • All input values are integers.

思路

类似于 C 题的思路, 预处理出 pre (从 1∼i1 \sim i1∼i 子段不同整数个数之和) 和 bak (从 i∼ni \sim ni∼n 子段不同整数个数之和) 数组。

接下来，我们面临的问题是：在 C 题中，只有一个切割点，我们可以在 O(n)\mathcal O(n)O(n) 的时间内求得答案。但是本题中，存在两个位置的切割点，O(N2)\mathcal O(N^2)O(N2) 的时间复杂度，难以快速求解答案，无法满足我们的需求。

我们需要先固定一个切割点(右切割点)：

记 中间子段 不同整数个数的和 为 f(l, r)

因此，题目转化为了求

max⁡j∈[2,n−1]{pre1+f(2,j), pre2+f(3,j), pre3+f(4,j), … , prej−1+f(j,j)}+ bakj+1\max\limits_{j\in[2, n-1]} \{pre_1 + f(2,j),\ pre_2 + f(3, j),\ pre_3+f(4, j),\ \dots\ ,\ pre_{j-1}+f(j,j)\} \\ +\ bak_{j+1} j∈[2,n−1]max​{pre1​+f(2,j), pre2​+f(3,j), pre3​+f(4,j), … , prej−1​+f(j,j)}+ bakj+1​

观察上面的式子，当 jjj 变为 j+1j+1j+1 时，

若 v[j+1]v[j + 1]v[j+1] 存在于中间子段， 则其加入无法对中间子段的不同整数的个数产生贡献。

而当 v[j+1]v[j + 1]v[j+1] 不存在于中间子段时， 则其加入将对中间子段不同整数个数产生贡献。

因此，我们需要记录当前位置的 viv_ivi​ 上一次出现的位置 (若前面未曾出现，则为 000) 。因此定义 lst 数组 记录我们所需信息。

我们再看回 pre1+f(2,j), pre2+f(3,j), pre3+f(4,j), … , prej−1+f(j,j)pre_1 + f(2,j),\ pre_2 + f(3, j),\ pre_3+f(4, j),\ \dots\ ,\ pre_{j-1}+f(j,j)pre1​+f(2,j), pre2​+f(3,j), pre3​+f(4,j), … , prej−1​+f(j,j) 序列：（将 pre 简称为 p）

可以发现，不仅仅是 f(l, r) 右边界扩展了，序列还增加了一项 prej+f(j+1,j+1)pre_{j}+f(j+1,j+1)prej​+f(j+1,j+1) 。

G - Maximize Distance

题目描述

给定一个有向图，包含 NNN 个顶点和 MMM 条边。顶点编号为 1,2,…,N1, 2, \dots, N1,2,…,N。第 jjj 条边（j=1,2,…,Mj=1, 2, \dots, Mj=1,2,…,M）从顶点 uju_juj​ 指向顶点 vjv_jvj​。保证顶点 NNN 可以从顶点 111 到达。

初始时，所有边的权重均为 000。我们从 MMM 条边中选择恰好 KKK 条，将它们的权重修改为 111。请你求出在修改权重后的图中，从顶点 111 到顶点 NNN 的最短距离的最大值。

Constraints

  • 2≤N≤302 \leq N \leq 302≤N≤30
  • 1≤K≤M≤1001 \leq K \leq M \leq 1001≤K≤M≤100
  • 1≤uj,vj≤N1 \leq u_j, v_j \leq N1≤uj​,vj​≤N
  • uj≠vju_j \neq v_juj​=vj​
  • In the given graph, vertex NNN is reachable from vertex 111.
  • All input values are integers.

思路

网络流是图论中的一类问题，研究如何在一个有向图中分配流量。图中的边有容量限制，流量不能超过容量。核心问题包括：

  1. 最大流问题：从源点 sss 到汇点 ttt 的最大流量。
  2. 最小割问题：找到一组边，使得删除这些边后 sss 和 ttt 不连通，且这些边的总容量最小。
Dinic 算法

Dinic 算法是求解最大流的高效算法，时间复杂度为 O(V2E)O(V^2 E)O(V2E)，核心步骤如下：

  1. 分层图：通过 BFS 构建从源点出发的分层图，记录每个节点的层级。
  2. 阻塞流：通过 DFS 在分层图上寻找增广路径，并更新剩余容量。
问题转换

最短距离的定义是路径上所有权重之和的最小值。我们需要选择 KKK 条边设为 111，使得所有可能的路径中，权重之和的最小值尽可能大。这个问题可以通过二分答案解决：

  • 假设答案为 ddd，需要验证是否存在一种选边方式，使得所有从 111 到 NNN 的路径的权重之和至少为 ddd。
  • 若存在，尝试更大的 ddd；否则，尝试更小的 ddd。
网络流建模

为了验证某个 ddd 是否可行，构造一个分层图，将问题转化为最大流问题：

  1. 分层图构造：

    • 将每个顶点拆分为 ddd 层，每层代表路径中的一个步骤。顶点 uuu 在第 kkk 层的节点编号为 u+(k−1)×Nu + (k-1) \times Nu+(k−1)×N。
    • 对每条原图的边 (uj,vj)(u_j, v_j)(uj​,vj​)，在第 kkk 层添加两条边：
      • 层内边：从第 kkk 层的 uju_juj​ 指向第 kkk 层的 vjv_jvj​，容量为 111（表示选中这条边）。
      • 跨层边：从第 kkk 层的 uju_juj​ 指向第 k+1k+1k+1 层的 vjv_jvj​，容量为极大值（如 10610^6106，表示不选中这条边）。
    • 在每层末尾添加边：从当前层的 NNN 指向下一层的 NNN，容量极大值（确保可以跨层到达终点）。

  2. 最大流的意义：

    • 从第 111 层的起点 111 到第 ddd 层的终点 N×dN \times dN×d 的路径必须经过 ddd 步。
    • 每条路径中使用的层内边数量即为选中的边数量。最大流的值即为需要选中的边的最小数量。
    • 若最大流 ≤K\leq K≤K，则说明可以用 ≤K\leq K≤K 条边构造分层图，保证原图中所有路径的权重至少为 ddd。
二分答案

通过二分法确定最大的 ddd：

  • 初始范围：l=0l = 0l=0（最短距离下限），r=N−1r = N-1r=N−1（路径长度的上限）。
  • 验证步骤：
    1. 计算中点 mid=⌊l+r+12⌋mid = \lfloor \frac{l + r + 1}{2} \rfloormid=⌊2l+r+1​⌋。
    2. 构造分层图并计算最大流。
    3. 若最大流 ≤K\leq K≤K，则令 l=midl = midl=mid；否则令 r=mid−1r = mid - 1r=mid−1。
分层图示例

假设 d=2d = 2d=2，原图中有一条边 (1,2)(1, 2)(1,2)。分层图构造如下：

  • 第 111 层：顶点 111 和 222。
  • 第 222 层：顶点 333 和 444（对应原顶点 111 和 222）。
  • 边 (1,2)(1, 2)(1,2) 的处理：
    • 层内边：1→21 \rightarrow 21→2，容量 111。
    • 跨层边：1→41 \rightarrow 41→4，容量 10610^6106。
复杂度分析
  • 二分次数：O(log⁡N)\mathcal O(\log N)O(logN)。
  • 每次 Dinic 的时间：O(d×M)\mathcal O(d \times M)O(d×M)。
  • 总复杂度：O(d×M×log⁡N)\mathcal O(d \times M \times \log N)O(d×M×logN)，在合理范围内。
总结

通过将问题转化为分层图的最大流问题，并利用二分法确定答案，可以在合理时间内求解最短距离的最大值。关键在于理解分层图的构造和网络流模型的建立，将原问题中的选边操作映射为网络中的流量限制。最终答案即为二分过程中找到的最大可行 ddd。
